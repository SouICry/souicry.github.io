Excel SQL

Excel PivotTable SQL Migrater

.NET
C#
SQL
Closed Source

During my summer 2015 internship at Applied Medical, I created this program to automate the migration of Excel PivotTable sources from an old database to a new one. Since the tables are used in every day business, it was crucial they immediately function correctly after update.

These pivot tables each contained a connection string that specified how to connect to the database, and a SQL command string to query the data. Each file could contain multiple of these connections as well.

Updating the connection strings was relatively easy, but modifiying the SQL queries was quite complicated. Many commands contained multiple nested queries, and I could not find an open source library that could accurately parse it. Not only did table names change, but many tables were merged, views were removed and added, and certain columns were also removed or added.

In the end, I wrote a custom SQL parser that analyzed the structure of the SQL query, distinguishing all major components as specified in the T-SQL standard.

It also analyzed each pertinent part, including the flow of all the different join types (left, right, full, inner, outer, cross, etc) as well as the convoluted naming schemes of tables, columns and aliases, many of which were repeated and conflicted with each other.

With the information from the analysis, table merges and column changes were applied, and logic modifications to the AND statement were made. Table naming conventions were standardized, and other parts of the code were also cleaned up.

The program functioned wonderfully, correctly migrating all the Excel tables.

The source code totaled around 5000 lines, with the parsing and editing logic spanning over 2000 lines excluding comments.

Unfortunately, I am not permitted to release the source code to the public. You can contact my manager about it however; just email me a request for the contact info.

JSObject

JSObject for Unity C#

C#
Unity
GitHub

While I was working on a cross platform game in Unity in C#, I ran into a problem. My team intended to store game data in JSON format, but since Unity ships with the Mono 2.0 compiler, there were no built in dynamic data types to hold JSON objects (dynamic type was introduced in C# 4.0). While open source parsers were available, to avoid have each sub-team build their own custom data interface logic, I created a small library called JSObject as a drop in replacement for JavaScript style objects.

The JSObject type accepts any valid JavaScript type implicitly, storing properties as Dictionaries and arrays as List. I intended it to function as any var would, implicitly supporting all standard operator and data types, while keeping track of its current type and correctly implicitly converting to standard C# types. Support for the open source JSONObject parser is provided through the JSONParser class I created.

Example usage:

Random rnd = new Random(); JSObject characters = JSONParser.parseFile("characters.json"); JSObject john = characters["John"]; if (john["age"] >= 18){     String message = john["happyMessages"][rnd.next(0, john["happyMessages"].Count());     Face face = new Face(john["happyFace"]);     return new Action(message, face); } else {     return new Action(null, new Face(john["sadFace"]); }
